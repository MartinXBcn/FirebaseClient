Firebase Cloud Messaging Web Client

Instruction

1. Place the index.html and auth.php files in your web server or install the http web server locally in your PC e.g. using XAMMP, UiformServer, phython and Node.js and open this HTML file with the web browser through the web server URL e.g. http://localhost/index.html.

2. Specify the information below which you can get it from your web app in the project settings here

The VAPID key can be taken from the Web Push certificates section under Project settings > Cloud Messaging

The Client Email and Private Key are taken from the Service Account key file and only required for message sending purpose.

3. Click the Initialize & Get Token button to initialize your Firebase Cloud Messaging Web Client and get the registration token to test with your Arduino device.

You also have to allow or enable notification for your web server site URL in your web browser settings or pop-up.

After the registration token was generated, you can now use for the registeration token in your Arduino code.

4. Click the Download SW File button to download the service worker javascript file that is required for background message notification.

Save the firebase-messaging-sw.js that downloaded to the same directory of this index.html file.

In case you can't download file, you can also create a new firebase-messaging-sw.js by click the Copy SW Content button and paste it to your firebase-messaging-sw.js file.

5. Reload this HTML page and click the Initialize & Get Token button again. Now your web client is ready to receive the messages.

6. To compose and send the message, this example app will send HTTP request to the googleapis server in the behalf of server admin.

The access token was used for authorization and obtained from sending the auth.php request.

The message to be sent should be valid HTTP v1 FCM messages format.
